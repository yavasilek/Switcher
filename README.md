# Switcher

Небольшая Windows-утилита в трее для автоматического исправления слов, набранных в неправильной русской или английской раскладке.

Switcher следит за последним набранным словом, распознает частые ошибки раскладки вроде `ghbdtn -> привет`, переключает активную раскладку, заменяет текст и проигрывает системный звук. Также есть ручная конвертация и откат последнего автоисправления.

## Возможности

- Автоисправление слов после пробела или знака препинания.
- Смена раскладки на нужную после исправления.
- Звук при автоматической и ручной смене.
- Откат последней автоматической замены.
- Ручная конвертация текущего или последнего слова.
- Ручная конвертация выделенного текста через буфер обмена с восстановлением старого содержимого.
- Пауза и список исключённых процессов для менеджеров паролей, терминалов и других приложений.
- Редактируемые пользовательские словари для русских и английских слов.
- История последних исправлений в окне настроек.
- Отдельный выбор системного звука для направлений `EN -> RU` и `RU -> EN`.
- Portable-запуск без установки.
- При portable-запуске Switcher спрашивает: установить в систему или продолжить portable-режим.
- Установка в профиль пользователя без прав администратора.
- Проверка новых версий на GitHub и установка обновления из приложения.
- Видимый прогресс скачивания и подготовки обновления.
- Тихая фоновая проверка обновлений без торможения старта.
- Окно настроек адаптировано под повышенный DPI и длинные русские строки.
- Настраиваемые горячие клавиши для ручной конвертации, отката и паузы.
- Импорт и экспорт настроек в JSON-файл.
- Подсказка первого запуска с основными командами.
- Расширенный встроенный словарь частых русских и английских слов.
- Редактор точечных автозамен, которые применяются до эвристики.
- Режим обучения: ручные исправления одиночных слов могут автоматически попадать в точечные замены.
- Диагностика активного окна, раскладки, последнего слова и причины решения.
- Проверяемая смена раскладки с несколькими попытками.
- Запасная замена текста через буфер обмена, если обычный `SendInput` не сработал.
- Список слов, которые Switcher никогда не должен исправлять.
- Смелый режим для коротких слов.
- Автоматическое восстановление keyboard hook после долгой тишины.
- Иконка в трее и простые настройки.
- Включение автозапуска вместе с Windows.

## Управление

- `Ctrl+Alt+Space` - конвертировать текущее слово или последнее слово перед курсором.
- `Ctrl+Alt+Enter` - конвертировать выделенный текст.
- `Ctrl+Alt+Backspace` - откатить последнее автоматическое исправление.
- `Ctrl+Alt+P` - включить или выключить паузу.
- Двойной клик по иконке в трее - открыть настройки.
- Меню трея - включить или выключить автопереключение, звук, паузу, автозагрузку, установку и проверку обновлений.

Комбинации можно изменить во вкладке `Настройки`. Чтобы программа не мешала обычному набору текста, каждая комбинация должна содержать `Ctrl` или `Alt`.

## Установка

Switcher можно использовать двумя способами:

- Portable: скачать `Switcher.exe` и запустить из любой папки.
- Установка в систему: открыть настройки, вкладка `Установка`, нажать `Установить в систему`.

При запуске portable-копии Switcher спрашивает, что сделать:

- установить в систему и запустить установленную копию;
- продолжить portable-запуск;
- закрыть приложение.

Установка выполняется без прав администратора в папку:

```text
%LocalAppData%\Programs\Switcher\Switcher.exe
```

После установки Switcher создаёт ярлык в меню Пуск и может прописать автозапуск. Удаление доступно в той же вкладке `Установка`.

## Обновления

Во вкладке `Установка` есть проверка новых версий через GitHub Releases. Если доступна новая версия, Switcher скачает свежий `Switcher.exe`, покажет прогресс загрузки, закроет текущий процесс, заменит файл и запустится снова.

То же самое можно сделать из меню трея через `Проверить обновления`.

Если установка обновления запущена из трея, Switcher сам откроет окно настроек, чтобы было видно этапы: скачивание, процент загрузки и подготовку к перезапуску.

Также Switcher осторожно проверяет обновления сам:

- проверка включена по умолчанию;
- стартует через несколько секунд после запуска;
- выполняется не чаще одного раза в 12 часов;
- использует короткий сетевой timeout;
- при ошибке сети ничего не показывает и не тормозит запуск;
- если найдена новая версия, показывает уведомление в трее.

## Перенос настроек

Во вкладке `Настройки` доступны:

- настройка горячих клавиш;
- экспорт настроек в JSON;
- импорт настроек из JSON;
- повторный показ подсказки первого запуска.

Экспорт сохраняет основные параметры, списки, историю, звуки, автозапуск и горячие клавиши. При импорте текущие настройки заменяются настройками из выбранного файла.

## Точечные автозамены

Во вкладке `Замены` можно добавить точные пары в формате:

```text
ghbdtn -> привет
зкщпкфь -> program
```

Такие правила применяются раньше обычной эвристики и полезны для коротких слов, имён, рабочих терминов и случаев, где автоматическое распознавание слишком осторожное.

Если включено обучение на ручной конвертации, Switcher добавляет в этот список ручные исправления одиночных слов. Выделенный текст не используется для обучения, чтобы не засорять правила фразами.

## Надёжность и диагностика

Во вкладке `Диагностика` видно:

- активный процесс и заголовок окна;
- текущая раскладка активного окна;
- последнее набранное слово;
- последняя пойманная клавиша;
- причина последнего решения: исправлено, пропущено, слово в исключениях, низкая уверенность;
- последняя смена раскладки и способ ввода текста;
- состояние keyboard hook и число его переустановок.

На вкладке `Основное` есть дополнительные режимы:

- `Смелее исправлять короткие слова` - разрешает более уверенные автозамены коротких слов.
- `Использовать буфер обмена...` - если обычный ввод через `SendInput` не прошёл, Switcher попробует удалить старое слово и вставить новое через буфер, после чего восстановит старое содержимое буфера.
- `Автоматически переустанавливать keyboard hook...` - помогает после сна Windows или долгой работы, если hook перестал получать клавиши.

На вкладке `Списки` есть колонка `Никогда не исправлять`. Слова оттуда не будут автозаменяться даже если эвристика считает замену подходящей.

## Сборка

```powershell
dotnet build -c Release
dotnet publish -c Release -r win-x64 -o .\publish\win-x64 -p:PublishSingleFile=true -p:SelfContained=true -p:EnableCompressionInSingleFile=true -p:DebugType=None -p:DebugSymbols=false
```

Готовый файл после публикации:

```text
publish\win-x64\Switcher.exe
```

## Автоматическая сборка на GitHub

В репозитории настроены GitHub Actions:

- `.github/workflows/build.yml` собирает проект и загружает `Switcher.exe` как артефакт при push и pull request.
- `.github/workflows/release.yml` собирает один Windows x64 exe-файл и прикрепляет его к GitHub Release при отправке тега вида `v0.1.0`.

Создать новый релиз локально:

```powershell
git tag v0.8.0
git push origin main --tags
```

## Проверка

```powershell
Start-Process .\publish\win-x64\Switcher.exe -ArgumentList '--self-test' -Wait -PassThru -WindowStyle Hidden
```

Self-test проверяет базовые пары вроде `ghbdtn -> привет`, `руддщ -> hello`, `ntcn -> тест`, `ьн -> my`, `рш -> hi`, пользовательский словарь, точечные автозамены, список слов без исправления, конвертацию текста с пробелами и сохранение настроек в JSON.
